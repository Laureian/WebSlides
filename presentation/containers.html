VM vs Docker

Czym jest Docker?

  • Api do obsługi contenerów

    zarządza kontenerami linuxowymi

  • Microservice/wątek/funkcjonalność

    kontener można porównać do wątku

  • Posiada swoje zasoby

    pamięć, miejsce, czas procesora, adres ip etc.

  • Jeden przepis na środowisko

    łatwe wdrożenia na różne środowiska

  • Wyizolowane środowisko

    kontenery nie mają na siebie wpływu

  • Skalowalny

    reakcja na zwiększony ruch - nowe kontenery

Zalety dlaczego świat oszalał?

  • Ułatwia proces developmentu

    Łatwiejsze wdrożenia, jedno spójne środowisko

  • Skalowalne

    Ponieważ kontenery są lekkie, można szybko stworzyć odpowiednią ilość instancji

  • Wspiera microservices

    świetnie się nadają do ich budowania

  • Posiada wersjonowanie

    trywialne rollbacki

  • Można zagnieżdżać dockery

    docker w dockerze, w dockerze docker...

Wady dlaczego nie jest tak kolorowo?

  • Sam docker nie nadaje się na produkcję

    potrzebne są orchestratory. Inaczej stawiamy apkę na środowisku developerskim inaczej na prod.

  • Monolity - niekoniecznie

    Docker został stworzony raczej do niewielkich serviceów. Idealnie kiedy kontener reprezentuje jakąs pracę do wykonania.

  • Wymaga, innego podejścia do programowania

    kontenery mogą się ze sobą komunikować ale trzeba pamiętać, że powinny to być zamknięte funkcjonalności

  • Kontener może wyłożyć Docker Engine

    i wszystko leży

  • Debugowanie

    zmora
